## Роль в проекте

В проекте я выступал в роли **Backend-разработчика и технического лидера (TechLead)**.  
Моя основная задача заключалась в архитектурной проработке системы, декомпозиции приложения на модули, подготовке use cases для каждого компонента и реализации ключевых backend-модулей.

---

## Перечень задач и описание выполненных мною работ

### Анализ требований
Проанализировал требования к проекту интернет-магазина, выделил основные пользовательские сценарии (регистрация, авторизация, работа с товарами, отзывы, продвижения,  корзиной, заказами) и зафиксировал их в виде use cases.

### Декомпозиция системы на модули
На основе требований выполнил декомпозицию системы на логические домены. Для каждого модуля были определены границы ответственности и основные функции, что позволило выстроить понятную структуру проекта и упростить параллельную работу команды.

### Подготовка use cases
Для каждого домена был подготовлен список use cases, описывающих ожидаемое поведение модуля. Эти use cases стали основой для дальнейшей реализации и помогли участникам команды лучше понять назначение своих компонентов.

### Проектирование архитектуры приложения
Спроектировал архитектуру backend-приложения в формате модульного монолита. Был выбран подход с разделением по доменам и внутренним слоям (API, Service, Repository).

### Проектирование структуры проекта
Была разработана единая структура проекта, ориентированная на домены, а не на технические слои. Такой подход позволил каждому участнику работать в рамках своего модуля, минимизируя пересечения и конфликты в коде.

### Проектирование модели данных 
Была разработана информационная модель данных для всего проекта в ввиде ER-диаграммы.  

### Реализация модулей
Лично мной были реализованы следующие модули:
- **core** — базовая инфраструктура проекта, конфигурация, работа с БД;
- **users** — управление пользователями, профилями и ролями;
- **auth** — регистрация, аутентификация, авторизация, работа с access и refresh токенами.

Другие модули реализовывались участниками команды самостоятельно в рамках выделенных им доменов.

## Использованные технологии и инструменты

В ходе работы над проектом мной использовались следующие технологии и инструменты:

- **Python** — основной язык разработки.
- **FastAPI** — фреймворк для создания асинхронного REST API.
- **Pydantic** — валидация данных и описание схем.
- **SQLAlchemy (async)** — ORM для работы с PostgreSQL.
- **Alembic** — управление миграциями базы данных.
- **Pytest** — написание автотестов для API.
- **Docker** — контейнеризация приложения и базы данных.
- **Docker Compose** — управление окружениями.
- **Uvicorn** — ASGI-сервер для запуска приложения.

## Саморефлексия

### Достижения
К своим основным достижениям в проекте могу отнести:
- декомпозицию системы на модули и формирование понятной архитектуры;
- подготовку use cases для каждого домена;
- реализацию базовых и критически важных модулей backend-приложения;
- практический опыт работы в роли технического лидера.

### Трудности
Наибольшие сложности возникли при работе с технической частью проекта:
- настройка автотестирования с использованием отдельной базы данных для тестов;
- изменение конфигурации приложения для корректной работы в режимах `dev` и `test`;
- работа с миграциями базы данных (Alembic), в том числе восстановление корректной цепочки миграций после удаления промежуточного файла.

Эти задачи потребовали дополнительного времени, но позволили глубже разобраться в устройстве backend-приложений и процессах их сопровождения.

### Общие впечатления от командной работы
Проект дал полезный опыт командной разработки backend-приложения. Работу над групповым проектом считаю полезной.
