# Отчет о проделанной работе по модулю ORDERS

1) Моя роль:
Backend-разработчик,  задача -  реализовать модуль orders и включить его в работу проекта.

3) Описание работы:
Реализовал модуль управления заказами для интернет-магазина. Создал полный цикл обработки заказов: создание из корзины, просмотр, обновление статусов, удаление. Система интегрирована с другими модулями: пользователями, каталогом товаров и корзиной.
Модуль включает 5 основных файлов:
1. models.py - ORM-модели Order и OrderItem для работы с базой данных
2. schemas.py- Pydantic схемы для валидации входных/выходных данных  
3. repository.py - слой доступа к БД с CRUD операциями
4. service.py - бизнес-логика и обработка заказов
5. api.py - REST API эндпоинты с авторизацией
6. test_orders.py - автотесты для проверки функционала
Основная логика: пользователь создает заказ из корзины, система проверяет наличие товаров и остатки, фиксирует цены, создает заказ и очищает корзину. Заказ проходит статусы: pending → processing → shipped → delivered. 
Пользователи видят только свои заказы, админы — все. Удалять можно только отмененные заказы.

3) Работал со следующими технологиями:
Язык - Python 3.12.
FastAPI - для создания REST API с автогенерацией документации OpenAPI.
SQLAlchemy  - ORM для работы с базой данных.
Pydantic - для валидации данных. Создал схемы с валидаторами для полей, использовал Field для ограничений (min_length, max_length), Config для работы с ORM.
PostgreSQL + ENUM - для хранения данных и статусов заказов. Использовал enum для статусов заказа.
Alembic - для миграций базы данных. Создал миграцию для таблиц orders и order_items.
Pytest + aiohttp - для написания автотестов. Реализовал тесты на создание заказов, проверку прав доступа, обработку ошибок.
Авторизация - для защиты эндпоинтов. Интегрировал с модулем auth, использовал зависимости для проверки ролей (user/admin).
Работал в VS Code, использовал Poetry для управления зависимостями, Git для контроля версий. Писал код с учетом принципов SOLID, DRY.

5) Итог:
- Реализован один из ключевых модулей проекта  - orders.
- Получен практический опыт в решении проблем, связанных с инфраструктурой и тестированием.
