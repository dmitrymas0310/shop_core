# Отчёт о работе над проектом shop_core

Моя роль: 
Backend-разработчик (Python/FastAPI), участник команды разработки сервиса онлайн-магазина.
Перечень моих задач с описанием работы над ними

    Реализация модуля отзывов (reviews)
    — Разработал полноценный CRUD-модуль для управления отзывами пользователей на товары.
    — Реализовал модели данных, Pydantic-схемы, CRUD-операции и роутеры в соответствии с архитектурой проекта.
    — Обеспечил защиту эндпоинтов: только автор отзыва может его редактировать или удалять, остальные действия требуют аутентификации.
    Написание и отладка автотестов
    — Написал интеграционные тесты для модуля reviews с использованием aiohttp и pytest.
    — Реализовал фикстуры для регистрации/авторизации пользователей и создания тестовых данных.
    — Диагностировал и устранял ошибки в тестах: 500 Internal Server Error, 409 Conflict, ScopeMismatch.
    Командное взаимодействие
    — Активно участвовал в код-ревью, обсуждении архитектурных решений и синхронизации с другими разработчиками.
    — Делился находками по устранению ошибок.

Использованные технологии и инструменты

    Язык и фреймворк: Python 3.12, FastAPI, SQLAlchemy (async), Pydantic v2  
    База данных: PostgreSQL, Alembic (миграции)  
    Контейнеризация: Docker, docker-compose  
    Тестирование: pytest, aiohttp, async fixtures  
    Инструменты: Poetry (менеджер зависимостей), Uvicorn (ASGI-сервер), Git/GitHub  
    Подходы: REST API, JWT-аутентификация, Repository-паттерн (в стиле проекта), TDD (test-driven development)

Саморефлексия

Достижения:
Успешно реализовал один из ключевых модулей проекта — отзывы — с полным покрытием автотестами.
Научился эффективно диагностировать сложные ошибки, связанные с инфраструктурой (миграции, Docker) и асинхронным тестированием.
Получил практический опыт работы в команде: синхронизация через GitHub, разделение задач, совместная отладка.

Трудности:
- Столкнулся с конфликтом портов PostgreSQL (локальный сервер vs Docker) — решил путём остановки локального сервиса.
- Затруднения с ScopeMismatch в pytest-фикстурах — разобрался в особенностях работы pytest-asyncio и aiohttp.
- Ошибки 500 при отсутствии таблиц в БД — научился читать логи сервера и применять миграции корректно.

Общие впечатления:
Работа над проектом shop_core стала ценным опытом применения современных практик backend-разработки. Командный формат позволил быстрее находить решения, делиться знаниями и поддерживать высокое качество кода. Особенно полезным оказалось углубление в асинхронное тестирование и управление миграциями. Проект помог закрепить навыки работы с FastAPI, Docker и PostgreSQL в реальных кейсах.
